Задание 1. Дипломатическая миссия
Студент создал класс DiplomaticMission и фабрику DiplomaticMissionFactory. В программе эти классы не используются, потому что Program.cs заменён на RandomMissionFactory. По заданию требовалось добавить дипломатическую миссию в список фабрик в Program.cs и увидеть её вывод наряду с остальными. Студент этого не сделал. В коде миссии используется const int DefaultDiplomaticDuration = 20. Лучше заменить на private static readonly, так как константа встраивается в IL и при изменении требует перекомпиляции зависимых сборок. Кроме того, строка отчёта "Торговые соглашения заключены успешно" не содержит имени миссии, что нарушает единообразие с другими миссиями (например, в CombatMission отчёт начинается с имени). Рекомендую придерживаться общего формата: $"{Name}: ...". Хорошо, что конструктор задаёт Name и Duration, но лучше вынести значения по умолчанию в readonly поля.

Задание 2. Спасательная миссия
Студент создал RescueMission и RescueMissionFactory, длительность вычисляется в зависимости от peopleCount. В Program.cs фабрика не добавлена в список (как требовалось в задании 2), но используется внутри RandomMissionFactory. Это допустимо, но задание явно просило добавить её в Program.cs. В коде RescueMission есть const int BaseRescueDuration и PeoplePerExtraDay, а также магическое число 5 и 10. По уникальным замечаниям, const следует заменить на readonly. Также имя миссии "Спасательная миссия" задаётся прямо в конструкторе – лучше вынести в приватное статическое поле. Целочисленное деление peopleCount / PeoplePerExtraDay может отбрасывать остаток, что допустимо по условию, но стоит учитывать. В остальном реализация корректна: Execute выводит сообщение, GetReport возвращает строку с количеством спасённых.

Задание 3. Случайная миссия
Студент создал RandomMissionFactory, который хранит список всех фабрик и случайным образом выбирает одну. Это отличный пример композиции. Однако в конструкторе RandomMissionFactory используется const int DefaultRescuePeopleCount = 50. По замечанию, const следует заменить на private readonly int, а имя поля изменить на _defaultRescuePeopleCount (стиль для приватных полей). В Program.Main() создаётся только RandomMissionFactory, и выводится одна случайная миссия. Программа соответствует условию: при каждом запуске миссия случайна. Но студент полностью проигнорировал требования заданий 1 и 2 о выводе всех миссий через список фабрик. Это критическое упущение. Также в Program.cs нет цикла, но в задании это не строго требовалось.

Общие замечания
- Использование const вместо readonly. Для значений, которые не являются универсальными константами (например, длительность миссии, количество людей), readonly предпочтительнее, так как const встраивается в код и при изменении значения потребуется перекомпиляция всех ссылающихся сборок. Пример: private readonly int defaultPeopleCount = 50;
- Магические числа в отчётах: "500 тонн", "12 кораблей", "5 систем". Лучше вынести в поля с понятными именами (например, _cargoAmount = 500). Это улучшит читаемость и упростит изменение.
- Значения Name и Duration задаются в конструкторе напрямую. Стоит вынести их в статические поля (private static readonly string DefaultName = "Грузовая миссия") и использовать в конструкторе. Это централизует значения по умолчанию.
- Execute() у всех миссий выводит на консоль. Пожелание из замечаний – возвращать строку, оставляя вывод на усмотрение вызывающего кода. В учебном проекте допустимо, но для тестируемости лучше разделять логику и вывод.
- Единообразие страдает: в DiplomaticMission отчёт не содержит имени, в других – содержит. В одних методах есть эмодзи (🤝, 🚀), в других – нет. Старайтесь придерживаться единого стиля.
- В Program.cs используется internal static void Main() – это допустимо, но обычно public static void Main.
